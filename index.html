Sunspot

Solr-powered search for Ruby objects

All the power of the Solr search engine; all the beauty of Ruby. Sunspot exposes all of Solr's most powerful search features using an API of elegant DSLs. That means robust, flexible fulltext search with no boolean queries and no string programming.

Index your objects.

{% highlight ruby %} class Post < ActiveRecord::Base searchable do text :title, :body text :comments do comments.map { |comment| comment.body } end integer :blog_id integer :author_id integer :category_ids, :multiple => true time :published_at string :sort_title do title.downcase.gsub(/^(an?|the)/, '') end end end {% endhighlight %}

Search for them.

{% highlight ruby %} Post.search do fulltext 'best pizza' with :blog_id, 1 with(:published_at).less_than Time.now order_by :published_at, :desc paginate :page => 2, :per_page => 15 facet :category_ids, :author_id end {% endhighlight %}

Use it with or without Rails.

The sunspot_rails gem integrates Sunspot into Rails with drop-in ease, extending ActiveRecord objects for searchability and managing the commit cycle transparently. sunspot_rails works with Rails 2.3 and Rails 3.0. Get Solr search in your Rails app in five minutes.

If you're not using Rails, or you're using Rails without ActiveRecord, the core Sunspot library can be hooked into your persistence layer with just a few lines of code.

Get all this.

  • Define searchable fields in an intuitive DSL, either indexing data directly from attributes or defining virtual attributes for search-specific information.
  • Use Sunspot's search DSL to restrict results using all the usual logical operators, negate them, or exclude individual objects from the results.
  • Harness Solr's powerful Dismax fulltext tool kit, including index-time and search-time field boost, phrase fields for extra-precise scoring, and boost queries for custom search-time relevance calculation.
  • Use Solr highlighting to show your users why you're returning the results you are. Custom deferred formatting allows your view layer to decide how to display the highlight.
  • Index subclasses, search for superclasses: Sunspot stores the entire class hierarchy for each object that's indexed.
  • Use field facets for easy, powerful drill-down search. If the field contains foreign keys, automatically get access to the referenced model in each facet row.
  • Use query facets to define search refinement on arbitrary filters.
  • Compose fulltext and geographical searches, calculating result scores using both proximity and keyword relevance.
  • If you've got WillPaginate installed, use it seamlessly with Sunspot's search results.
